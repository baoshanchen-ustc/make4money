## 1. 背景与目标

### 1.1 背景

当前平台用户增长依赖自然流量和付费投放，获客成本较高。通过建立 Affiliate 分销体系，利用现有用户的社交网络进行裂变传播，降低获客成本，同时激励 KOL 合作推广。

### 1.2 目标

| 目标 | 指标 | 预期 |
|------|------|------|
| 降低获客成本 | CAC | 降低 30% |
| 提升用户裂变 | 邀请转化率 | ≥ 15% |
| KOL 渠道建设 | 活跃 KOL 数 | 首季度 20+ |

### 1.3 非目标

- 本期不做多级分销（仅一级）
- 本期不做社交平台自动分享
- 本期不做终身绑定佣金上限（后续根据数据决定）

## 2. 用户角色

### 2.1 角色定义

| 角色 | 说明 | 特权 |
|------|------|------|
| **普通用户** | 所有注册用户 | 可生成邀请链接，获得基础分成 |
| **推广达人** | 达到阶梯门槛的用户 | 更高佣金比例 |
| **KOL/合伙人** | 后台人工设置 | 专属推广码、最高比例、可发优惠券、可设终身绑定 |

### 2.2 角色升级路径

```
普通用户 ──(有效邀请达标)──→ 推广达人
                              ↓
                        (人工审核设置)
                              ↓
                          KOL/合伙人
```

## 3. 功能需求

### 3.1 邀请机制

#### 3.1.1 邀请码/链接生成

| 项目 | 说明 |
|------|------|
| 邀请链接 | 每个用户唯一，格式：`{domain}/r/{code}` |
| 邀请码 | 6-8 位字母数字，支持自定义（KOL） |
| 二维码 | 基于邀请链接生成，支持下载 |
| 分享海报 | 预设模板，含二维码和利益点 |

#### 3.1.2 邀请绑定规则

- 新用户注册时绑定邀请关系
- 绑定后**不可更改**
- 同一设备/IP 24 小时内限制注册 N 个账号（防刷）

#### 3.1.3 注册奖励

| 触发条件 | 邀请人奖励 | 被邀请人奖励 |
|----------|-----------|-------------|
| 被邀请人完成注册 | +X 余额 | +X 余额 |

> X 为后台可配置金额

### 3.2 佣金分成

#### 3.2.1 分成模式

| 模式 | 说明 | 适用对象 |
|------|------|----------|
| **首充分成** | 被邀请人首次充值时，邀请人获得一次性佣金 | 默认所有用户 |
| **终身绑定** | 被邀请人每次充值，邀请人均获得佣金 | KOL/合伙人（后台设置） |

#### 3.2.2 分成门槛

邀请人需满足以下条件之一，才能获得佣金分成：

- 已购买过包月套餐，**或**
- 历史消费累计满 Y 金额

> Y 为后台可配置金额

#### 3.2.3 阶梯佣金

根据**有效邀请人数**，佣金比例递增：

| 档位 | 有效邀请数 | 佣金比例 |
|------|-----------|----------|
| 档位 1 | 0 ~ A 人 | P1% |
| 档位 2 | A+1 ~ B 人 | P2% |
| 档位 3 | B+1 人以上 | P3%（上限） |

> A、B、P1、P2、P3 均为后台可配置
>
> **有效邀请**定义：被邀请人完成首充

#### 3.2.4 佣金计算公式

```
佣金 = 被邀请人充值金额 × 当前佣金比例
```

### 3.3 KOL 体系

#### 3.3.1 KOL 专属功能

| 功能 | 说明 |
|------|------|
| 专属推广码 | 可自定义，如 `KOLDAXIN` |
| 专属佣金比例 | 独立设置，不受阶梯限制 |
| 终身绑定 | 可为 KOL 开启终身分成 |
| 优惠券发放 | KOL 专属优惠券模板，用户使用后享优惠 |

#### 3.3.2 用户优惠

通过 KOL 推广码注册的用户可获得：

- 注册额外奖励（后台配置）
- 首充折扣/满减券

### 3.4 提现系统

#### 3.4.1 提现规则

| 项目 | 规则 |
|------|------|
| 提现门槛 | 累计佣金 ≥ $100（可配置） |
| 提现方式 | 待定（支付宝/银行卡/USDT） |
| 审核流程 | 人工审核 |
| 到账时间 | 审核通过后 T+3 |

#### 3.4.2 佣金状态

```
待确认 ──(被邀请人充值后7天无退款)──→ 已确认 ──(申请提现)──→ 提现中 ──→ 已提现
```

### 3.5 推广中心（用户端）

#### 3.5.1 页面结构

```
推广中心首页
├── 收益概览
│   ├── 累计收益
│   ├── 可提现金额
│   └── 提现进度条（距 $100 还差 $XX）
├── 推广数据
│   ├── 邀请人数（总/有效）
│   └── 当前档位 & 进度
├── 邀请工具
│   ├── 复制链接
│   ├── 生成海报
│   └── 二维码下载
├── 邀请记录
│   └── 列表（昵称、注册时间、状态、贡献佣金）
└── 提现记录
    └── 列表（金额、时间、状态）
```

### 3.6 后台管理

#### 3.6.1 配置管理

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 注册奖励金额 | 数值 | 邀请人/被邀请人各得多少余额 |
| 分成门槛-包月 | 开关 | 是否要求购买过包月 |
| 分成门槛-消费额 | 数值 | 消费满多少可获得分成 |
| 阶梯档位配置 | JSON | 人数区间 + 对应比例 |
| 佣金比例上限 | 数值 | 阶梯最高比例 |
| 提现门槛 | 数值 | 最低提现金额 |
| 佣金确认周期 | 数值 | 充值后多少天确认佣金 |

#### 3.6.2 用户管理

- 查看用户分销数据
- 设置用户为 KOL/合伙人
- 为 KOL 配置专属参数
- 设置终身绑定

#### 3.6.3 数据报表

- 分销概览（邀请数、有效数、佣金总额）
- KOL 排行榜
- 提现申请列表

## 4. 用户故事

### US-001：生成邀请链接

```
作为 普通用户
我想要 生成我的专属邀请链接
以便 分享给朋友获得奖励

验收标准：
- [ ] 点击「复制链接」可复制到剪贴板
- [ ] 链接格式：{domain}/r/{code}
- [ ] 链接有效且可正常跳转
```

### US-002：注册获得奖励

```
作为 被邀请人
我想要 通过邀请链接注册后获得余额奖励
以便 体验平台服务

验收标准：
- [ ] 通过邀请链接进入注册页
- [ ] 注册成功后，双方各获得配置金额的余额
- [ ] 余额实时到账，可在钱包查看
```

### US-003：查看推广数据

```
作为 邀请人
我想要 查看我的推广效果数据
以便 了解收益情况

验收标准：
- [ ] 显示累计收益、可提现金额
- [ ] 显示邀请人数（总数/有效数）
- [ ] 显示当前档位和升级进度
```

### US-004：获得充值佣金

```
作为 邀请人（已满足分成门槛）
我想要 在被邀请人充值时获得佣金
以便 增加我的收益

验收标准：
- [ ] 被邀请人充值后，佣金记录生成（待确认状态）
- [ ] 7天无退款后，佣金状态变为已确认
- [ ] 佣金金额 = 充值金额 × 当前比例
```

### US-005：申请提现

```
作为 邀请人
我想要 将累计佣金提现
以便 获得实际收益

验收标准：
- [ ] 可提现金额 ≥ 门槛时，显示提现按钮
- [ ] 提交提现申请，进入审核流程
- [ ] 可查看提现记录和状态
```

### US-006：KOL 专属推广

```
作为 KOL
我想要 使用我的专属推广码
以便 获得更高佣金并给用户优惠

验收标准：
- [ ] 可查看/复制专属推广码
- [ ] 用户使用该码注册，自动绑定关系
- [ ] 用户可获得专属优惠（券/折扣）
- [ ] KOL 按专属比例获得佣金
```

### US-007：后台配置分销规则

```
作为 管理员
我想要 在后台配置各项分销参数
以便 灵活调整运营策略

验收标准：
- [ ] 可配置注册奖励金额
- [ ] 可配置分成门槛条件
- [ ] 可配置阶梯档位和比例
- [ ] 可配置提现门槛
- [ ] 配置修改后实时生效
```

### US-008：设置 KOL

```
作为 管理员
我想要 将指定用户设置为 KOL
以便 开展 KOL 合作推广

验收标准：
- [ ] 可搜索并选择用户
- [ ] 可设置专属推广码
- [ ] 可设置专属佣金比例
- [ ] 可开启/关闭终身绑定
- [ ] 可关联优惠券模板
```

## 5. 数据指标

### 5.1 核心指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 邀请转化率 | 有效用户 / 注册用户 | ≥ 15% |
| 人均邀请数 | 总有效邀请 / 活跃邀请人 | ≥ 3 |
| 佣金成本率 | 佣金支出 / 被邀用户充值额 | ≤ 8% |
| 提现率 | 提现金额 / 累计佣金 | 监控 |

### 5.2 监控告警

- 单日注册奖励支出超 $X → 告警（防刷）
- 单用户贡献佣金超 $Y → 标记（终身绑定监控）
- KOL 转化率低于 Z% → 提醒运营

## 6. 风险与对策

| 风险 | 对策 |
|------|------|
| 羊毛党批量注册 | 设备指纹 + IP 限频 + 手机验证 |
| 自充自提 | 禁止自己邀请自己，充值后冷却期 |
| KOL 低质量流量 | 设置最低转化率要求 |
| 佣金纠纷 | 明确规则，保留完整日志 |
